Ejercicio 2 — Callbacks

filter · map · reduce · Lección 2

Recuerda: callbacks en métodos de array

Un callback es una función que pasas como argumento. Los métodos de array la llaman una vez por cada elemento para saber qué hacer con él. 

// .filter(callback) → retorna nuevo array solo con los que pasen la prueba
// .map(callback)    → retorna nuevo array transformando cada elemento
// .reduce(callback, valorInicial) → reduce el array a un único valor

const nums = [1, 2, 3, 4];
nums.filter(n => n > 2);            // [3, 4]   — el callback retorna true/false
nums.map(n => n * 10);              // [10,20,30,40] — retorna la transformación
nums.reduce((acc, n) => acc + n, 0); // 10       — acc acumula, retorna el nuevo acc

🧪 Parte A — filter()

filter recorre el array y retorna un nuevo array con los elementos donde tu callback retorna true. El array original no cambia.
1
Escribe el callback dentro de productos.filter(). El callback recibe cada producto como parámetro — llámalo p.
2
El callback debe retornar true o false. Para quedarnos con los disponibles, retorna la propiedad p.disponible — ella ya es un booleano.
3

// COMPLETA el callback:
const disponibles = productos.filter( /* escribe aquí tu callback */ );

// Resultado esperado: [ Laptop, Mouse, Teclado ]

🧪 Parte B — map()

map transforma cada elemento y retorna un nuevo array del mismo tamaño. Si entran 5 objetos, salen 5 transformaciones.
1
Escribe el callback dentro de productos.map(). Recibe cada producto p.
2
El callback debe retornar solo el nombre de cada producto. Accede a él con p.nombre.
3
El resultado es un array de 5 strings, uno por cada producto.
// COMPLETA el callback:
const nombres = productos.map( /* escribe aquí tu callback */ );

// Resultado esperado: [ "Laptop", "Mouse", "Monitor", "Teclado", "Webcam" ]

🧪 Parte C — reduce()

reduce acumula los elementos en un único valor. El callback recibe (acumulador, elemento) y debe retornar el nuevo acumulador. El segundo argumento de reduce es el valor inicial del acumulador.
1
Primero filtra los disponibles (ya lo hiciste en la Parte A). Usa la misma lógica.
2
Escribe el callback de reduce. Recibe (acc, p). Debe retornar acc + p.precio para ir acumulando.
3
Después del callback pon , 0 — ese es el valor inicial del acumulador. Si lo omites, puede dar error con arrays vacíos.
4
